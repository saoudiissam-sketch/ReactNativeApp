Architecture Proposée pour l'Application Mobile (React Native) L'objectif est de créer une base de code unique, maintenable et performante pour iOS et Android, tout en réutilisant la logique métier existante.

Philosophie et Stack Technologique Framework : React Native avec Expo. Expo simplifie considérablement le développement, la gestion des builds et les mises à jour OTA (Over-The-Air). Il gère nativement une grande partie des configurations complexes. Langage : TypeScript, pour maintenir la robustesse et la cohérence avec le projet web. UI & Theming : NativeWind est le choix logique pour traduire l'esthétique de Tailwind CSS vers React Native. On peut également envisager une bibliothèque de composants comme Tamagui pour des performances optimisées et un theming avancé. Navigation : React Navigation est le standard de la communauté. Nous utiliserons une combinaison de Bottom Tab Navigator pour les sections principales et de Stack Navigator pour les flux internes. Gestion d'état : La structure actuelle avec React Context est viable pour le prototype. Pour une scalabilité à long terme, une migration vers Zustand est recommandée. C'est une solution légère, simple et performante qui évite le "boilerplate" de Redux. Persistance des données : localStorage n'existe pas en React Native. Nous utiliserons MMKV de Shopify. C'est une bibliothèque de stockage clé-valeur extrêmement rapide et chiffrée, bien plus performante que l'alternative standard AsyncStorage.
